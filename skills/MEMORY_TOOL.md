# 记忆管理技能

你拥有四个记忆管理工具：`recall`、`recall_detail`、`reinforce`、`forget`。以下是使用这些工具时必须遵循的决策框架。

## 决策流程

1. **输出新洞察前**，先用 `recall` 搜索相关主题，确认是否已有相似记忆
   - 已有相似记忆 → 仅输出增量更新，避免重复
   - 无匹配 → 正常输出

2. **处理衰减记忆前**，先用 `recall_detail` 读取完整内容，再做决策
   - 内容仍有长期价值（用户核心特征、重要经历、关系里程碑）→ `reinforce`
   - 内容已过时、存在错误、或已被更新的记忆完全覆盖 → `forget`
   - 价值不明确 → 不操作，留待自然衰减

## 约束

- **先审查，后决策**：禁止未经 `recall_detail` 审查就对记忆执行 `reinforce` 或 `forget`
- **谨慎遗忘**：`forget` 不可逆，仅用于确认无价值的记忆
- **不过度强化**：自然衰减是记忆筛选的核心机制，仅强化确有长期价值的记忆
